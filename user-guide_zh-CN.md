# 点对点交易平台 - 用户手册

# 1. 概述

点对点交易平台（下文简称P2PE）是一款完全去中心化的交易DApp。

P2PE具备如下优势与特点：

1）	大额订单交易0手续费；

2）	自由限定交易价格和数量；

3）	支持任意自定义的WRC20标准token交易；

4）	基于HTLC（哈希时间锁定合约）和Token Approve开关双重安全保障；

5）	订单未上链前可随时取消，取消后订单不再可用，无需长时间等待；

6）	订单交易信息不公开，仅在链数据上可见；

7）	无需充值，不需要将token充到任何在线钱包或合约地址中，可在ledger中直接操作；

8）	通过跨链可支持多链多币种交易。

# 2. 应用场景

Lucy想要用0.0248 BTC的单价购买1,000个ETH，计划总花费24.8个BTC。

这个操作如果在交易所中完成，需要耗费大量的时间，并且最终平均价格和成本不可控。

因此Lucy在社交网络上发帖：《单价0.0248 BTC求购1,000个ETH》

Tom看到了Lucy的帖子，想要与她完成交易，于是他们双方沟通完毕后，按照如下方法进行操作，在很短的时间内，就安全高效的通过wanchain的跨链机制和P2PE产品完成了本次交易。

# 5. 常见问题与回答

**问题1**：P2PE相比以太坊上普通的HTLC交易产品有什么优势？

**回答**：P2PE相比以太坊上普通的HTLC交易产品有如下4点优势：

1）	运行在Wanchain上，与跨链机制结合，可在同一个智能合约中实现多链多币种交易，减少信任成本；

2）	交易过程中无需将token充值或转入合约地址中，提高了安全性和可控性；

3）	交易双方使用同一个智能合约交易，合约代码公开透明，保证安全；

4）	交易发起用户PartyA可PartyB完成交易之前随时取消交易，取消无延迟；

**问题2**：P2PE为什么要用户手动点击approve开关对智能合约执行资产授权操作，而不是后台自动完成？

**回答**：因为授权开关是用户交易的一重安全保障，用户可随时通过手动关闭授权开关来取消交易，保障资产不受损失。另一方面，因为approve操作权限较高，理应由用户知情和主动发起。

**问题3**：使用P2PE交易时是不是完全没有手续费？

**回答**：使用P2PE进行WRC20代币交易时，完全没有手续费。但如果用户需要先跨链再交易，则需要支付少量的跨链手续费。

**问题4**：P2PE产品完全开源吗？能否移植到以太坊工作？

**回答**：智能合约代码和前端页面代码完全开源，可以移植到以太坊工作，但移植后将不具备跨链功能。未来在wanchain双向桥跨链功能完成后，也可在以太坊上执行跨链交易。但交易手续费比wanchain上更贵。

**问题5**：我在P2PE上开启approve后，什么时候可以关闭？

**回答**：只要交易成功完成，就可以关闭了。

**问题6**：P2PE和wandex有什么关系？

**回答**：P2PE是独立的DApp，运行不依赖wandex，但其链上结算部分使用的是wandex的智能合约。这个智能合约并不限制使用者，任何人开发的DApp均可使用其完成链上结算交易。
